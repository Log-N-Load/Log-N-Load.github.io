Ataque Smart Car


Este artículo describe el proceso de ataque de repetición (Replay Attack) llevado a cabo contra el Freenove 4WD Smart Car. Se utilizó un ataque MITM (Man-in-the-Middle) para capturar los paquetes de comunicación entre el dispositivo de control y el vehículo, permitiendo así replicar comandos legítimos para manipular su comportamiento.

Captura y Análisis de Tráfico
Ataque MITM con EtterCap
Se ejecuta un ataque MITM entre el dispositivo controlador y el coche utilizando EtterCap. Esto permite interceptar y modificar los paquetes de datos enviados entre ambos dispositivos.

Análisis de paquetes con Wireshark
Usando Wireshark, se analizan los paquetes de comunicación en busca de aquellos que contienen comandos relevantes, como la activación del motor o el encendido de los LEDs.

Filtrado por IP y puerto
Aplicando filtros en Wireshark para las direcciones IP de los dispositivos y el puerto de comunicación (5000), se identifican transmisiones específicas de activación de LEDs y control de movimiento.

Creación del Script de Ataque
Una vez se tiene la información necesaria, se procede a la creación de un script en Python para hacer el ataque replay. El script se conecta a la máquina, enciende las luces, activa los motores y cada cierto tiempo cambia el color de los LED hasta que recibe una orden de parada.

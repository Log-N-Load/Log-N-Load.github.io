HTML Injection - Reflected (GET)

Se trata de inyectar código HTML malicioso en la URL (por ejemplo, como parámetro GET), y el servidor refleja este contenido en la página que el usuario ve.
El código puede ejecutarse en el navegador del usuario, comprometiendo su seguridad.

  • Vector

El login es vulnerable a ataques tipo HTML Injection. Al introducir cualquier input, no lo sanitiza y lo muestra por pantalla.

  • Vulnerabilidad

La vulnerabilidad reside en que, el input no es sanitizado una vez lo ha recibido.

  • Solución/Mitigación

Es posible solucionar esta vulnerabilidad de varias formas, una de ellas sería utilizando POST en lugar de GET, pero en este caso se ha decidido utilizar funciones de php para quitar todos los tags y limpiar el input recibido.

HTML Injection - Stored (Blog)

En este tipo de ataque, el código HTML malicioso se almacena en el servidor (por ejemplo, en un comentario de un blog), y luego se muestra a otros usuarios que visitan la página, pudiendo ejecutar scripts dañinos en sus navegadores.

  • Vector

La web tiene un textbox que permite escribir un mensaje, al publicarlo se guarda en la base de datos y lo muestra en la sección de blog junto con el nombre del creador y la fecha de creación. Es vulnerable a ataques tipo HTML Injection.

  • Vulnerabilidad

La vulnerabilidad reside en que, el input no es sanitizado una vez lo ha recibido.

  • Solución/Mitigación

Es posible solucionar esta vulnerabilidad de varias formas, una de ellas sería mediante preparedstatement, pero en este caso se ha decidido utilizar funciones de php para quitar todos los tags y limpiar el input recibido.

Al atacar de nuevo, muestra solo el mensaje, pero no tiene los tags de HTML.

##

Tras solucionar el código y volver a atacar, se puede comprobar que en la URL muestra una cosa (el input que hemos enviado) y en la web HTML otra (el input después sanitizarlo).

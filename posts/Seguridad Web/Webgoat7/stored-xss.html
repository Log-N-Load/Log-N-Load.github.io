Stored

En este tipo de ataque, el código JavaScript malicioso se inyecta en una aplicación web y se almacena en el servidor (por ejemplo, en una base de datos), para ser ejecutado posteriormente en los navegadores de otros usuarios que visiten esa página.

En este caso, se añade un script que haga saltar la alerta “message” cuando se cargue.

En este caso, la aplicación ha guardado la entrada del blog y cada vez que se carga muestra la alerta. Es posible modificar el ataque para robar cookies de sesión por ejemplo.

Blind String SQL Injection {#blind-string-sql-injection}

Blind SQL String Injection ocurre cuando un atacante inyecta consultas SQL maliciosas en entradas de texto, sin recibir respuestas directas, pero deduciendo información a través de cambios en el comportamiento de la aplicación.

  • Vector de ataque:

El input devuelve “Valid” si es True o “Invalid” si es False.

Si se concatena o se añaden más condiciones a la query sigue devolviendo valores booleanos.

Haciendo pruebas se puede concatenar un select y deducir donde se almacena en la base de datos los valores que queremos obtener.

Con la siguiente query seleccionamos la primera letra del usuario.

Utilizando BurpSuite Intruder se puede repetir la consulta con todas las letras del alfabeto y comprobar cuales dan un resultado correcto.

Se añaden los campos “Valid” e “Invalid” para comprobar las letras correctas en el ataque.

Una vez sabemos la primera letra, se repite para averiguar la segunda, así hasta que no devuelva ningún resultado correcto.

Sacamos que el usuario es “Jill”.

  • Vulnerabilidad.

El servidor no limpia o comprueba el input que ha pasado el usuario, permitiendo este tipo de ataques.

  • Solución-Prevención.

Ya que solo debería permitir introducir 3 dígitos, se hace una comprobación, si el input tiene una longitud mayor a 3 solo usa los 3 primeros caracteres del input.

Si se introduce una cadena mayor a 3.

Devuelve “Invalid account number”
